# Математическая очередь с ZK-верификацией на Fortran

## Что это?

Это демонстрационный проект, показывающий возможности Fortran в современной криптографии. 
Реализована система очередей сообщений (похожая на Kafka) с Zero-Knowledge доказательствами.

## Что такое Zero-Knowledge (ZK)?

Zero-Knowledge доказательство - это математический способ доказать, что ты знаешь секрет,
**не раскрывая сам секрет**. 

Представь: ты хочешь доказать, что знаешь пароль, но не хочешь его говорить.
ZK-доказательство позволяет это сделать!

## Как это работает?

1. **Очередь сообщений** - как очередь в магазине, первым пришел - первым ушел (FIFO)
2. **ZK-доказательство** - каждое сообщение имеет математическое доказательство, что отправитель знает секретный ключ
3. **Верификация** - можно проверить валидность доказательства, не зная секретного ключа

## Математика (упрощенно)

Используется протокол Schnorr:
- Отправитель знает секретный ключ `x`
- Публичный ключ: `y = g^x mod p` (все знают)
- Доказательство: `(r, s)` где `r = g^k mod p` и `s = k - c*x mod (p-1)`
- Проверка: `g^s * y^c = r mod p`

## Компиляция и запуск

```bash
# Компиляция
gfortran -o zk_queue main.f90

# Запуск
./zk_queue
```

## Возможности Fortran для криптографии

✅ **Быстрые математические операции** - Fortran оптимизирован для вычислений
✅ **Работа с массивами** - эффективная обработка больших объемов данных
✅ **Параллелизм** - можно использовать OpenMP для ускорения
✅ **Точность вычислений** - важна для криптографии
